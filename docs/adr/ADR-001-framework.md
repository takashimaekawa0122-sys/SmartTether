# ADR-001: フレームワーク選定

- **日付**: 2026-03-22
- **ステータス**: 決定済み

## 決定事項
**Flutter** を採用する（React Native / ネイティブ開発は不採用）

## 理由
- iOS・Android両対応を1つのコードで実現する必要がある
- BLEバックグラウンド処理がReact Nativeより安定している
- flutter_reactive_bleプラグインが実績豊富
- LEDフラッシュ・バックグラウンド録音など深いシステム連携が必要

## 却下した選択肢
- **React Native**: BLEバックグラウンドが不安定。ネイティブブリッジが多くなる
- **Swift + Kotlin（ネイティブ）**: 開発コストが2倍になる。個人プロジェクトに不向き
