# ADR-006: Mi Fitness代替方針

- **日付**: 2026-04-05
- **ステータス**: 決定済み

## 決定事項
Smart TetherをXiaomi Smart Band 9の**メインコンパニオンアプリ**として設計し、Mi Fitnessを完全に不要にする。

## 背景・経緯
- iOSのBLE仕様により、1つのBLEデバイスに接続できるアプリは同時に1つのみ
- Mi FitnessとSmart Tetherを共存させると、使うたびに切り替えが必要になる
- 切り替え運用は現実的ではないため、Smart Tether一本化を決定

## 実装する機能と優先順位

| 優先度 | 機能 | 依存 | 難易度 |
|---|---|---|---|
| 🔴 最優先 | BLE認証（SPPv2） | なし | 高 |
| 🔴 最優先 | 置き忘れ防止（RSSI） | 認証 | 中 |
| 🔴 最優先 | ステルス操作 | 認証 | 中 |
| 🟡 次フェーズ | 時刻同期 | 認証 | 低 |
| 🟡 次フェーズ | 通知転送 | 認証 | 高 |
| 🟡 次フェーズ | 歩数・心拍表示 | 認証 | 中 |
| 🟢 後回し可 | 天気送信 | 認証 | 中 |
| 🟢 後回し可 | 睡眠データ | 認証 | 高 |
| ❌ 対象外 | ファームウェア更新 | — | 非常に高 |

## 実装方針
- プロトコルの参照実装として**Gadgetbridge**（オープンソース）のコードを活用する
- 認証（SPPv2）が動作することを確認してから、各機能を順番に追加する
- 一度にすべてを実装しない。動くものを積み上げていく方式

## Auth Key取得について
- Band 9リセット後は新しいAuth Keyが生成される
- 取得方法: Android端末にGadgetbridgeをインストール → Band 9をペアリング → デバイス設定でAuth Keyを確認
- 取得できたらSmart Tetherの設定画面に入力する（以降は不要）

## 却下した選択肢
- **Mi Fitnessと共存**: BLE排他制約により実現不可能
- **Mi FitnessにSmart Tetherを組み込む**: Mi Fitnessはクローズドソースのため不可能
- **別のデバイスを使う（AirTag等）**: ステルス操作機能が失われる
